1/ CPR description:
1/.1. CPR/Project Summary:

- the current tpldauftrag archives closed orders from kdauftrag to akdauftrag.

- normally the process will be started during the night before the next business day

- in emergency case the process need to be started during day but then it also archives closed order from the current business day, so business cannot see those orders from same day anymore and “blind” for the day.

1/.2. CPR/Project Scope:

- the application and start parameters should be verified and reworked that it will be recognized, that only previous business day closed orders will be archived.

challenge: if normal start e.g. at 23:30, then the same business day closed orders should be archived. if start is late, e.g. 8:00 next morning, then previous business day should be used only.
